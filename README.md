1. Установите необходимые инструменты
   Прежде чем начать работу с проектом, нужно убедиться, что у вас установлены следующие инструменты:

PHP 8.1+ (для backend)

Composer (для установки зависимостей PHP)

Node.js и npm/yarn (для frontend)

MySQL или другая база данных (для backend)

2. Настройка Backend (Laravel)
   Клонируйте репозиторий с проектом:

Откройте терминал и выполните команду:

bash
Copy
Edit
git clone https://github.com/IvanTaranenko/intent_solution.git
cd back-end
Настройка .env файла:

В корне проекта найдите файл .env.example и переименуйте его в .env. Настройте параметры базы данных:

.env

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=название_базы_данных
DB_USERNAME=пользователь_базы_данных
DB_PASSWORD=пароль_базы_данных
Убедитесь, что вы настроили правильный API токен для администратора:

ADMIN_TOKEN=суперсекретный_токен

Установите зависимости с помощью Composer:

В терминале выполните команду:


composer install
Запустите миграции для базы данных:

Создайте таблицы в базе данных:

php artisan migrate
Добавьте тестовые данные (опционально):

Запустите сидеры для добавления тестовых заявок:


php artisan db:seed --class=RequestSeeder
Запустите сервер Laravel:

Запустите локальный сервер:

bash
C
php artisan serve
Сервер будет доступен по адресу http://localhost:8000.

3. Настройка Frontend (React)
   Перейдите в папку с frontend проектом:
Если у вас есть отдельная папка для frontend, выполните команду:


cd front-end
Настройка .env файла для frontend:

Создайте или отредактируйте файл .env и укажите API URL вашего backend, а также токен администратора:


VITE_REACT_APP_API_URL=http://localhost:8000/api - либо ваш локальный сервер бекенда
VITE_REACT_APP_ADMIN_TOKEN=суперсекретный_токен   - он  должен совпадать с токеном 

Установите зависимости с помощью npm/yarn:

Если у вас используется npm, выполните команду:


npm install

Запустите frontend сервер:

После установки зависимостей запустите проект:


npm run dev
Ваш frontend будет доступен по адресу http://localhost:5173  

4. Использование приложения
   Для пользователей (неавторизованные):
   Отправка заявки: Пользователь может отправить заявку через форму на frontend.

Для администратора (с токеном):
Просмотр заявок: Администратор может просматривать список заявок через API.

Изменение статуса заявки: Администратор может изменить статус заявки через API.

Удаление заявки: Администратор может удалить заявку через API.


5. Примечания
   Авторизация:
   Все защищенные маршруты API требуют токен в заголовке Authorization: Bearer <ваш_токен>.

API (можно использовать через Postman или Insomnia):

POST /api/requests — Создать заявку
Пример запроса:
POST http://127.0.0.1:8000/api/requests/
Заголовок:
Content-Type: application/json
Authorization: Bearer superadmin
Тело запроса:

json

{
"name": "99999",
"email": "thejohnshow558@gmail.com",
"message": "dsadsadasdasd",
"status": "new"
}
GET /api/requests — Получить список заявок (только для администраторов)

PATCH /api/requests/{id} — Изменить статус заявки (только для администраторов)

DELETE /api/requests/{id} — Удалить заявку (только для администраторов)

Взаимодействие с приложением в React
Форма заявки:
Перейдите на http://localhost:5174/, где будет отображена форма для создания заявки. Заполните все поля и отправьте заявку.

Список заявок:
Перейдите на http://localhost:5174/list-of-requests.
При попытке доступа к этому разделу вам будет предложено ввести токен администратора (тот же токен, который используется в заголовке Authorization для защищенных API маршрутов). Токен необходим для выполнения операций (просмотр, изменение статуса, удаление).

Каждое действие, связанное с изменением данных, также потребует ввода токена администратора.
